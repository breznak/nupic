38 /**
39 * @b Responsibility
40 * The Exception class is the standard Numenta exception class.
41 * It is responsible for storing rich error information:
42 * the filename and line number where the exceptional situation
43 * occured and a message that describes the exception.
44 *
45 * @b Rationale:
46 * It is important to store the location (filename, line number) where
47 * an exception originated, but no standard excepton class does it.
48 * The stack trace is even better and brings C++ programming to the
49 * ease of use of languages like Java and C#.
50 *
51 * @b Usage:
52 * This class may be used directly by instatiating an instance
53 * and throwing it, but usually you will use the NTA_THROW macro
54 * that makes it much simpler by automatically retreiving the __FILE__
55 * and __LINE__ for you and also using a wrapping LogItem that allows
56 * you to construct the exception message conveniently using the <<
57 * streame operator (see nta/utils/Log.hpp for further details).
58 *
59 * @b Notes:
60 * 1. Exception is a subclass of the standard std::runtime_error.
61 * This is useful if your code needs to interoperate with other
62 * code that is not aware of the Exception class, but understands
63 * std::runtime_error. The what() method will return the exception message
64 * and the location information will not be avaialable to such code.
65 *
66 * 2. Source file and line number information is useful of course
67 * only if you have access to the source code. It is not recommended
68 * to display this information to users most of the time.
69 */
